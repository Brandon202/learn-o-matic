Hyper Text Markup Language

HTML is a Hyper Text Markup Language. Its kind of like the base of an app or website. You use tags to create the code. Here's a simple code.
            <!DOCTYPE html>
<html>
    <head>
        <title>Learn-o-matic</title>
        <meta charset="UTF-8">
    </head>
    <body>
        <h1>Learn-o-matic</h1>
        <p>This is my Learn-o-matic</p>
    </body>
</html>
Let me take you through that piece of code, step by step. The first line shows that you are using HTML5, the latest version of HTML. The second line is the html tag. Everything else besides the DOCTYPE tag is in the html tag. Next is the head tag. The head tag has links for your stylesheets, scripts for your external scripts, style tag for style (see 2nd tab for info on that), title, and charset="UTF-8", showing that you represent any character in the Unicode standard. You end the head tag. Then you start the body tag. Usually, everything in the body tag shows something, unless you hid it with JS or jQuery. (See 3rd tab for info on that). In the body tag, I put the biggest heading, h1, and have the text Learn-o-matic. There are 6 headings, h1 to h6. h1 is the biggest and h6 is the smallest. Then you end the body and html tags. Sometimes, you have a script tag to include JS right before the end of the body tag. (Also see 3rd tab for info on that).

Cascading Style Sheets

CSS styles the HTML, such as changing the background-color, or padding. Here's an example that changes the background-color of the body.
body {
    background-color: blue;
}

The first line targets the element it wants to put style on. The second line changes the background-color of that element to blue. Line 3 ends the style.
There are also other stuff that you could do with CSS. Here's an example.
div < p {
    color: green;
}

That takes all the div elements, and then takes all the p elements in the div elements and colors them green. But there is also another thing called SCSS (Sassy CSS).
$cool-blue: #0099ff;
body {
    background-color: $cool-blue;
}

That was using a variable. Cool, right? You could use variables over and over again. Here's an advanced example with using variables.
$cool-color: Aqua;
$cool-font: Pacifico;
div {
    background-color: $cool-color;
    p {
        font-family: $cool-font;
    }
}

That was using nesting. That saves a lot of time and energy. This is the same using CSS.
div {
    background-color: Aqua;
}
div < p {
    font-family: Pacifico;
}

SCSS is less time-consuming and way cooler. But you need to install SCSS on your computer and put it in a file ending with ".scss". Then put it in a stylesheet link. So if you're not on a computer, CSS is the way to go.

JavaScript

They say JS is used to make the output interactive. But JS can do so many incredible things without the user interacting. First, I'm going to teach you about comments. JS has 3 types of comments. They have the single-line comment.
// Some code...
The multi-line comment.
/*
Ta-da!
Ta-da!
*/

And the other cool comment. (Try it. It looks better on code editor).
/**
Ta-da!
@What_code_is_about
Details
  *
  *
  *
**/

Now I'm going to put JS in a script tag and change all p elements to say Hello!
<!DOCTYPE html>
<html>
    <head>
        <title>Learn-o-matic</title>
        <meta charset="UTF-8">
    </head>
    <body>
        <p>Learn-o-matic</p>
        <script>
            document.getElementByTagName("p").textContent = "Hello!";
        </script>
    </body>
</html>

I could've added an id named learn to the element and did document.getElementById("learn").textContent = "Hello!";
But imagine if you could add html tags, also. Well, you can! Here's a div with the id div. I'm going to put an h1 in it.
document.getElementById("div").innerHTML = "<h1>Oh yeah!</h1>";
Pretty sweet, huh? But you can do that even faster.
$("#div").html("<h1>Oh yeah!</h1>");
Thats called jQuery!
